⚙️ Supabase Setup

One-time setup — about 5 minutes. After this, your whole team will share the same live data.

1
Create a free Supabase account

Go to supabase.com → click Start your project → sign in with GitHub or Google.

2
Create a new project

Click New project. Give it a name (e.g. inventory), set a password, choose region East US or Canada, then click Create new project. Wait ~1 minute for it to start up.

3
Create the database tables — copy this SQL

In your Supabase project → left menu SQL Editor → New query. Paste the code below and click Run:

-- Inventory table create table if not exists items ( id text primary key, name text, loc text, qty integer, avail integer, status text, updated_at timestamptz default now() ); -- Damage reports table create table if not exists damages ( id bigint primary key, item_id text, item_name text, loc text, person text, severity text, description text, photos text, resolved boolean default false, created_at timestamptz default now() ); -- Movement log table create table if not exists log_entries ( id bigint primary key, created_at timestamptz default now(), type text, item_id text, item_name text, loc text, qty integer, person text, notes text, severity text, photos text ); -- Enable real-time updates alter publication supabase_realtime add table items; alter publication supabase_realtime add table damages; alter publication supabase_realtime add table log_entries;
4
Copy your API keys

In Supabase → Project Settings (gear icon, bottom-left) → API. Copy:

• Project URL — looks like https://xxxx.supabase.co

• anon public key — the long string under Project API keys

These keys are saved in your browser only. Each team member will need to enter them once.
